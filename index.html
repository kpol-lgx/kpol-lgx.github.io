12-二叉搜索树

发表于 2021-10-22 分类于 algorithm

参考资料：算法导论

二叉搜索树上的基本操作花费的时间和树的高度成正比。因此应当尽可能减小树的高度。

什么是二叉搜索树

本质上是一个二叉树，每个节点包含五个属性：key，卫星数据，left，right，p 。中文翻译比较奇怪，个人理解 key 就是 键值对 中的键，卫星数据就是值。left ，对应左分叉，right 对应右分叉，p 是父指针（没有母指针），这三个倒是确定的。

key的分布很有特点，左子树中的每一个key都不会比父节点的key大，右子树则全部大于父节点的key。

递归搜索按序简单输出所有key:

这是中序遍历的伪代码。先序，中序，后序的区别在于子树的根的key在左右子树的key的哪个位置，中序会先输出左子树的全部key，然后是root的key，最后是右子树的key，对于子树的子树采用同样的方法。递归搜索。

对于每个节点，搜索过程都会调用自身两次：一次是左子树，一次是右子树。

定理 12.1

如果 x 是一棵有 n 个节点的子树的根，那么调用 INORDER-TREE-WALK(x) 需要 \(\Theta (n)\) 时间

证明 每个节点都需要访问因此总时间 $ T(n) = (n)$ 。

对于空树，只需要一个小的常数时间（因为 x \(\neq\) NIL），设为 c ，所以 \(T(0)=c\) 。

n > 0 ，假设 x 的左子树右 k 个节点，则右子树有 n-k-1 个节点。$T(n) T(k) + T(n-k-1) + d $, \(d\) 为常数 ，\(d\) > 0 。这个不等式表明了总用时的一个上限（不包括递归调用的用时）

现在要证 \(T(n) = O(n)\) ，需证 \(T(n) \leq (c+d)n+c\) 。n=0显然成立。n>0 ，有$ T(n) T(k)+T(n-k-1)+d=((c+d)k+c)+((c+d)(n-k-1)+c)+d=(c+d)n+c-(c+d)+c+d=(c+d)n+c$

得证。

存疑，n>0，使用了\(T(n) = (c+d)n+c\) ，得出\(T(n) \leq (c+d)n+c\) ，用结果证明结果是对的，证明没有说服力

Reminder: 线性神经元的误差曲面

Reminder是立体3D的，水平面上是各个权重的变化，有一个垂直轴z轴对应误差变化。对于具有平方误差的线性神经元，这个图像是一个二次的碗（立体的盆）。

多层，非线性神经元的误差曲面复杂的多。但是在局部，一个二次的碗的一部分通常可以作为一个非常好的近似。

误差曲面为二次的碗时全数据集训练的收敛速度

除非椭圆是圆，否则最速下降的方向不会指向最小值。梯度总是在我们需要较小的移动距离时很大，在需要较大移动距离时则相反。即使在非线性的多层网络局部，误差曲面也是二次的，因此同样的解决方法可以被使用。

随机梯度下降法

如果数据集高度冗余，那么前一半的梯度几乎和后一半相同。所以不必计算全部梯度，要更新权重，只需要计算前一半的梯度，然后用后一半计算再下一次的梯度，总之，轮流使用就对了。这种更新权重的方式的极端版本会在每个案例之后更新，被称为"online" 。

小批量通常比 online 好，更新权重时前者有更少的计算，而且，同时为多个案例计算梯度采用矩阵乘法是非常高效的，特别是在GPUs上。

两种学习算法

如果我们对于所有训练案例使用完全的梯度计算，这有相当多漂亮的方法加速学习。优化社区已经研究优化光滑非线性函数的普遍问题许多年了。不过多层神经网络并不是他们研究的问题的典型，因此他们的方法需要大量的修改。

对于有非常大且冗余度高的训练集的大型神经网络，最好的办法就是使用小批量学习。

SIR-作业

发表于 2021-10-20 更新于 2021-10-22 分类于 数学建模（选修）作业

SIR型传染病模型的假设：

  1. 研究时间段内群体固定，总人数不发生变化，没有出生，死亡，迁入，迁出

  2. 病人接触易感人群后会按照一定概率传染。传染概率与传染病本身性质，感染人数和易感人数三者有关

  3. 疾病仅仅依靠人体自身免疫系统治愈，并且感染者痊愈后不会在感染

  4. 传染病对于易感人群的传染能力不会随时间变化

分析：

假设有一座城市，总人口为 M ，现在有某种传染病 Ct 在城内传播，初始易感人群为 S = M ，初始感染人群为 I = I0 ，初始痊愈人群为 R = 0。Ct 的感染系数为 \(\alpha\) ，人群的恢复系数为 \(\beta\) , \(\alpha\) 和 \(\beta\) 是常数。并且我们设不同人群的数量函数为

  1. 易感人群 S(t)
  2. 感染人群 I(t)
  3. 痊愈人群 R(t)

在模型假设中，单位时间内易感人群中感染 Ct 的人数为 \(\alpha S(t)I(t)\)，这是因为感染人数越多，可以同时接触感染者的易感人群也增加，感染速率提高；易感人数减少，会导致感染者不能接触到更多的易感者，因此感染速率下降。

与之不同的是，单位时间内感染人群中痊愈的人数仅仅有恢复系数和感染人数有关，即 \(\beta I(t)\)

显然有以下结论： \[ S(t) + I(t) + R(t) = M \]

以及： \[ \frac {dS(t)}{dt} = -\alpha S(t)I(t)\\ \\ \frac {dI(t)}{dt} = \alpha S(t)I(t) - \beta I(t)\\ \\ \frac {dR(t)}{dt} = \beta I(t) \]

(1)(2)为核心方程。

从(2)可知，易感人群始终不断减少，痊愈人群始终不断增加。

General recommended practies for AI

发表于 2021-09-21 更新于 2021-09-30

推荐训练

使用以人类为中心的设计方法

  • 设计内置功能时应当适当公开：清晰和可控对于良好的用户体验非常重要
  • 考虑增加和帮助：如果一个答案的概率很高并且能满足大量不同的使用者和案例，那么只给一个答案是合适的。在其他情况下， 系统提供多个选项给用户可能是最好的选择。技术上，从一个答案上要获得较高的精度肯定比多个答案要困难许多。
  • 在设计过程的早期对潜在的不利反馈进行建模，然后在全面部署之前对一小部分流量进行实时测试和迭代。
  • 与一组不同的用户和用户方案进行互动，并在项目开发之前和整个过程中纳入反馈。这将为项目构建丰富的用户视角，并增加受益于该技术的人数
