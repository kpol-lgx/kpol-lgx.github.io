The shell

发表于 2021-10-01 分类于 The missing semester

MIT The missing semester课程
原文

启动终端，出现命令提示符

missing 是机器名。 ~ 是home目录简写，也是此时所在目录。 $ 表示以非root身份登陆。

输入data后回车，显示时间。

echo 接受一个参数并打印结果。shell中空格用来分隔多个参数。如果输入文本中有空格，应该使用单引号或双引号，也可以使用\转义

shell本身自带部分命令。此外，环境变量中记录了一些程序的路径。shell会通过环境变量查找需要的程序。

显示环境变量。绝对路径，以:分隔。

查询echo所在路径

通过绝对路径调用echo,打印$PATH。这方法可以执行不在$PATH中的程序

通常，执行程序会在当前目录下操作，除非我们告诉它其他信息。

重定向
< file 和 > file

将echo的输出定向到hello.txt，，hello.txt代替屏幕成为标准输出

hello.txt 的内容作为输入进入cat

从左向右依次执行，先将hello.txt作为输入给到cat,然后cat的输出在送到hello2.txt

> file

会覆盖原有的数据，相当于删去原数据，写入新内容

>> file

追加模式，不会覆盖

|

管道。管道左边程序的输出会做为右边程序的输入，这可以同时使用多个工具处理任务。

挂载于/sys目录之下的是sysfs 文件系统。sysfs以文件形式公开了一部分内核参数

尽管使用了sudo，但是仍然显示请求被拒绝。这是因为|,>,和<是由shell执行的，
echo以及其他程序并不知道这些连接符，它们只是接受给自己的输入，然后输出。上面的例
子中，>将命令分为两部分：sudo echo 3和brightness。sudo将权限给了echo，
|由shell操作。shell尝试打开brightness文件夹但是没有权限。
