sed

发表于 2021-10-21 更新于 2021-10-22

Overview

通常以这种方式启动

sed SCRIPT INPUTFILE...

SCRIPT 脚本，INPUTFILE 输入文件

举例，将input.txt中的hello全部替换为world并且另存到output.txt

sed 's/hello/world/ input.txt > out.txt'

如果不指定INPUTFILE，或者INPUTFILE用 - 代替，sed会过滤标准输入，比如键盘键入，的内容。以下命令均等价：

sed将输出写到标准输出。使用 -i 选项在合适的地方修改文件而不是打印到标准输出。类似的，查W和 s//w 命令也会将输出写到其他文件。下面的命令将修改 file.txt文件而不产生任何输出

sed -i 's/hello/world/' file.txt

sed默认打印所有被处理的输入（不包括已经被像 d 这样的命令修改过的内容）。使用 -n 组织输出， p 命令打印指定行，以下命令仅仅打印输入文件的第45行。

sed -n '45p' file.txt

sed把多个输入文件当成一个长长的流。下面的例子打印第一个文件的第一行和最后一个文件的最后一行。使用 -s 反转这个行为。（译者注：怎么反转？反过来不一样吗？）

没有 -e 或 -f 选项时，sed使用第一个非选项的参数作为脚本 (script)，之后的非输入参量全部作为输入文件。如果 -e 或 -f 指定了脚本,所有非输入参量全部作为输入文件。选项 -e 和 -f 可以结合使用，并且能多次出现（在这种情况下最终生效的脚本是所有独立脚本的联合）
