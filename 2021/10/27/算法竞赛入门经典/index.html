第一章-程序设计入门

  1. 算法竞赛中不应当使用头文件 conio.h
  2. 每行输出均应以回车符结束，包括最后一行。除非特别说明，每行行首不应该有空格，但是行末通常可以有多余空格。另外，输出的每两个数或者字符串间应该以单个空格隔开。
  3. “明知故犯”是有益的：起码了解了发生错误的现象
  4. 尽可能使用 const 关键字声明常数。
  5. %03d 输出控制，以三位字符输出不足则在行首加0, 例如025。

三整数排序

先检查 a 和 b ，如果 a > b ，则交换 a 和 b ；接下来检查 a 和 c ，最后检查 b 和 c

第一次检查后必有 a <= b ，第二次检查后必有 a <= c ，这样就确定了最小值 a ，接下来只需要确定 b 和 c的大小即可。

2.4 算法竞赛中的输入输出框架

scanf函数有返回值，返回成功输入的变量的个数，当输入结束时，无法再次读取，返回0 。

在windows下输入完毕后先按Enter键，再按Ctrl-Z键，最后按Enter键，即可结束输入。在Linux下，按Ctrl-D键即可结束输入。

第一句定义符号LOCAL ，之后的 #ifdef ... #endif 代码只有在定义了LOCAL后才会被编译。这样提交时删去#define LOCAL 即可。更好的方法是在编译选项里定义LOCAL符号。

"-"表示这是编译选项，D意思是“定义(define)”，LOCAL 是被定义的符号。

如果想要把fopen版的程序改为读写标准输入输出，只需赋值"fin = stdin; fout = stdout; "即可，不要调用fopen和fclose来打开标准输入输出。

编写程序应当提高程序鲁棒性。

在多数据题目中，一个常见的错误是：计算完一组数据后某些变量没有重置，影响到下一组数据的求解。

而当嵌套的两个代码块中有同名变量时，内层变量会屏蔽外层变量，有时会引起十分隐蔽的错误。

使用编译选项 -Wall 编译程序时会给出许多（但不是全部）的警告信息。但是有些“错误”是合法的，只是不是所期望的。
